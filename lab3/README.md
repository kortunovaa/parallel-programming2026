## Реализация

В программу из лабораторной работы №1 были добавлены MPI функции для организации межпроцессного взаимодействия:

```cpp
int main(int argc, char* argv[]) {
    MPI_Init(&argc, &argv);

    int rank, num_processes;
    MPI_Comm_rank(MPI_COMM_WORLD, &rank);
    MPI_Comm_size(MPI_COMM_WORLD, &num_processes);

    // Только процесс 0 читает матрицы из файлов
    if (rank == 0) {
        MatrixA = Read_Matrix_From_File(matrixA_file);
        MatrixB = Read_Matrix_From_File(matrixB_file);
    }

    // Синхронизация процессов перед вычислениями
    MPI_Barrier(MPI_COMM_WORLD);
    
    // Процесс 0 выполняет умножение матриц
    if (rank == 0) {
        result = Multiply_Matrix_Sequential(MatrixA, MatrixB);
    }

    MPI_Finalize();
    return 0;
}
```

## 1. Программа из лабораторной работы №1 успешно модифицирована для параллельной работы с использованием технологии MPI

## 2. Корректность результатов подтверждена автоматической верификацией (столбец "Верно" = 1 для всех экспериментов)

## 3. Анализ производительности показал, что на одном компьютере MPI не дает ожидаемого ускорения:
1) Для матриц 200×200: время выполнения практически не меняется (195-202 мс)
2) Для матриц 400×400: 1 процесс показывает лучшее время (1675 мс) по сравнению с многопроцессными вариантами (1725-1752 мс)
3) Для матриц 800×800: минимальное время достигается при 1 процессе (13713 мс)

## 4. Полученные результаты демонстрируют принципиальное отличие MPI от OpenMP:
1) OpenMP работает через общую память (эффективно на одной машине)
2) MPI требует явного распределения данных между процессами для реального распараллеливания
3) Без распределения вычислений MPI добавляет только дополнительные накладные расходы
